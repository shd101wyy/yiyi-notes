# Git 文件命名的问题

当时在 Windows 下把一个文件由大写命名 `A.txt` 换为了小写命名 `a.txt` ，git 就发现了出了点问题，好像是两个文件依旧存在。尤其是在 VSCode 中依旧可以 Navigate 到 `A.txt`。  

当时我就很纳闷，今天终于在马老哥的指导下知道了原因。原来得设置 git 为

```bash
$ git config --global core.ignorecase false
```

这样就没问题了 👍
